### Week 4 Objectives:

- Understand real-time communication on AWS.
- Get familiar with Amazon API Gateway WebSocket and AWS Lambda.
- Practice building basic WebSocket API.

### Tasks to be carried out this week:

| Day | Task                                                                                                                  | Start Date | Completion Date | Reference Material                     |
| --- | --------------------------------------------------------------------------------------------------------------------- | ---------- | --------------- | -------------------------------------- |
| 2   | - Learn the differences between REST API and WebSocket API.<br>- Understand WebSocket use cases in real-time systems. | 11/05/2026 | 11/05/2026      | https://docs.aws.amazon.com/apigateway |
| 3   | - Create WebSocket API Gateway.<br>- Configure Routes ($connect, $disconnect, sendMessage).                           | 12/05/2026 | 12/05/2026      | AWS Documentation                      |
| 4   | - Connect WebSocket API with AWS Lambda.<br>- Handle connection and disconnection events.                             | 13/05/2026 | 13/05/2026      | https://docs.aws.amazon.com/lambda     |
| 5   | - Practice sending and receiving data via WebSocket.<br>- Store ConnectionId to manage user connections.              | 14/05/2026 | 14/05/2026      | AWS Documentation                      |
| 6   | - Check logs with CloudWatch.<br>- Handle errors during sending and receiving data.                                   | 15/05/2026 | 15/05/2026      | https://docs.aws.amazon.com/cloudwatch |

### Week 4 Achievements:

- Understand the principles of WebSocket API operation on AWS.

- Successfully create WebSocket API Gateway.

- Connect WebSocket with AWS Lambda.

- Know how to store and manage ConnectionId.

- Able to send and receive data in real-time.

- Use CloudWatch to monitor and handle errors.
